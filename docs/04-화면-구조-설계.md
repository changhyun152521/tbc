## 1. 화면 목록 정의

### 1.1 공통 (비인증)

| No | 화면 ID | 화면명 | 설명 |
|----|---------|--------|------|
| C01 | Login | 로그인 | loginId, password 입력. 로그인 성공 시 역할별 첫 화면으로 리다이렉트 |

### 1.2 관리자 (admin)

| No | 화면 ID | 화면명 | 설명 |
|----|---------|--------|------|
| A01 | AdminDashboard | 관리자 대시보드 | 학생 수, 반 수, 강사 수, 최근 활동 요약 |
| A02 | StudentList | 학생 목록 | 검색·필터(반, 학년). 테이블/카드 목록 |
| A03 | StudentDetail | 학생 상세 | 기본 정보, 반, 학부모 연결 정보 표시 |
| A04 | StudentForm | 학생 등록/수정 | 등록: 필수+선택 입력. 수정: 학생·학부모 ID/비밀번호 포함 |
| A05 | TeacherList | 강사 목록 | 강사 목록 |
| A06 | TeacherDetail | 강사 상세 | 강사 정보 + 담당 반 목록 |
| A07 | TeacherForm | 강사 등록/수정 | 이름, 로그인 ID, 비밀번호 |
| A08 | ClassList | 반 목록 | 반 목록 (학생 수, 강사 수 요약 가능) |
| A09 | ClassDetail | 반 상세 | 반 정보, 소속 강사 목록, 소속 학생 목록 |
| A10 | ClassForm | 반 생성/수정 | 이름, 설명. 상세에서 강사 지정 연동 |
| A11 | AdminProfile | 내정보 (관리자) | 로그인 ID, 비밀번호, 전화번호 변경 |

| No | 화면 ID | 화면명 | 설명 |

| T01 | TeacherDashboard | 강사 홈 | 담당 반 요약, 오늘/이번 주 수업·과제 요약 |
| T02 | MyClassList | 담당 반 목록 | 반 카드/리스트 → 반 선택 |
| T03 | ClassDetailForTeacher | 반 상세 (강사) | 해당 반 학생 목록, 진도·과제·테스트 요약 |
| T04 | LessonList | 수업 목록 | 반 선택 후 수업 목록. 기간 필터 |
| T05 | LessonForm | 수업 등록/수정 | 제목, 수업 일자, 과목, 순서 |
| T06 | ProgressInput | 진도 입력 | 수업 선택 → 학생별 진도 내용/완료 입력 |
| T07 | AssignmentList | 과제 목록 | 반별 과제 목록 |
| T08 | AssignmentForm | 과제 등록/수정 | 제목, 연동 수업, 마감일 |
| T09 | AssignmentCompletion | 과제 완료 체크 | 과제 선택 → 학생별 완료/미완료 토글 |
| T10 | TestList | 테스트 목록 | 반별 테스트 목록 |
| T11 | TestForm | 테스트 등록/수정 | 제목, 시험 일자, 과목, 만점 |
| T12 | TestScoreInput | 테스트 점수 입력 | 테스트 선택 → 학생별 점수 입력 |
| T13 | TeacherProfile | 내정보 (강사) | 로그인 ID, 비밀번호, 전화번호 변경 |

### 1.4 학생 (student)

| No | 화면 ID | 화면명 | 설명 |
|----|---------|--------|------|
| S01 | StudentDashboard | 학생 홈 | 내 진도 요약, 최근 과제, 최근 테스트 결과 한 화면 |

## 2. 역할별 화면 흐름도

### 2.1 공통 진입

```
[로그인 화면 C01]
       │
       │ 로그인 성공
       ├── role=admin  ──→ [관리자 대시보드 A01]
       ├── role=teacher ──→ [강사 홈 T01]
       ├── role=student ──→ [학생 홈 S01]
       └── role=parent  ──→ [학부모 홈 P01]
```

### 2.2 관리자 흐름

```
[A01 대시보드]
    │
    ├─ 학생 관리
    │     ├─ [A02 학생 목록] → [A03 학생 상세] → [A04 학생 수정]
    │     └─ [A04 학생 등록] (목록에서 진입)
    │
    ├─ 강사 관리
    │     ├─ [A05 강사 목록] → [A06 강사 상세] → [A07 강사 수정]
    │     └─ [A07 강사 등록]
    │
    ├─ 반 관리
    │     ├─ [A08 반 목록] → [A09 반 상세] (강사 지정, 학생 배정 연동)
    │     └─ [A10 반 생성/수정]
    │
    └─ [A11 내정보]
```

### 2.3 강사 흐름

```
[T01 강사 홈]
    │
    ├─ [T02 담당 반 목록] → [T03 반 상세]
    │                            │
    │                            ├─ 수업: [T04 수업 목록] → [T05 수업 등록/수정]
    │                            │         → [T06 진도 입력] (수업 선택 후)
    │                            ├─ 과제: [T07 과제 목록] → [T08 과제 등록/수정]
    │                            │         → [T09 과제 완료 체크]
    │                            └─ 테스트: [T10 테스트 목록] → [T11 테스트 등록/수정]
    │                                                  → [T12 테스트 점수 입력]
    │
    └─ [T13 내정보]
```

### 2.4 학생 흐름

```
[S01 학생 홈]
    │
    ├─ [S02 내 진도]
    ├─ [S03 내 과제]
    ├─ [S04 내 테스트 결과]
    └─ [S05 내정보]
```

- 학생은 홈에서 진도/과제/테스트를 탭 또는 카드로 진입. 3탭 이내 권장.

### 2.5 학부모 흐름

```
[P01 학부모 홈]
    │
    ├─ [P02 자녀 진도]
    ├─ [P03 자녀 과제]
    ├─ [P04 자녀 테스트 결과]
    └─ [P05 내정보]
```

- 학부모도 홈에서 자녀 진도/과제/테스트로 직행. 구조는 학생과 동일, 데이터만 “자녀 1명” 기준.

### 2.6 흐름도 요약 (Mermaid 스타일 텍스트)

```
flowchart TB
    Login[C01 로그인]

    Login --> S01[S01 학생 홈]
    Login --> P01[P01 학부모 홈]

    A01 --> A02[A02 학생목록]
    A02 --> A03[A03 학생상세]
    A03 --> A04[A04 학생등록/수정]
    A01 --> A05[A05 강사목록]
    A05 --> A06[A06 강사상세]
    A06 --> A07[A07 강사등록/수정]
    A01 --> A08[A08 반목록]
    A08 --> A09[A09 반상세]

    A01 --> A11[A11 내정보]

    T01 --> T02[T02 담당반목록]
    T02 --> T03[T03 반상세]
    T03 --> T04[T04 수업목록]
    T04 --> T06[T06 진도입력]

    S01 --> S02[S02 내진도]
```

### 3.6 라우트 가드

- `/login` 제외하고, 모든 역할 구간(`/admin`, `/teacher`, `/student`, `/parent`)은 해당 역할 로그인 사용자만 접근.
- 잘못된 역할로 URL 직접 접근 시: 로그인 페이지로 리다이렉트 또는 403 안내 후 본인 역할 첫 화면으로 이동.

## 4. 공통 컴포넌트·레이아웃

### 4.1 레이아웃

| 구분 | 설명 |
|------|------|
| AdminLayout | 관리자: 상단 네비(또는 사이드바), 학생/강사/반/내정보 메뉴. PC 위주. |
| TeacherLayout | 강사: 상단 또는 하단 메뉴. 홈, 반 목록, 내정보. 모바일·PC 공용. |
| StudentLayout | 학생: 하단 탭 또는 상단 메뉴. 홈, 진도, 과제, 테스트, 내정보. 모바일 우선. |
| ParentLayout | 학부모: 학생과 동일. 홈, 진도, 과제, 테스트, 내정보. 모바일 우선. |

### 4.2 공통 컴포넌트 (개념)

| 컴포넌트 | 용도 |
|----------|------|
| AppHeader | 로고, 타이틀, 로그아웃 등 |
| BottomNav | 학생/학부모 하단 탭 (홈, 진도, 과제, 테스트, 내정보) |
| Sidebar / TopNav | 관리자/강사 메뉴 |
| LoadingSpinner | 로딩 표시 |
| ErrorMessage | API 에러 메시지 표시 |
| FormInput, FormSelect | 폼 필드 (접근성·모바일 터치 고려) |

### 4.3 반응형

- 학생/학부모: 모바일 퍼스트. 320px~480px 기준. 하단 탭은 터치 영역 충분히.
- 관리자: 768px 이상에서 테이블·폼 가독성 확보. 모바일에서는 카드·리스트 위주로 전환 가능.
- 강사: 모바일에서 진도/과제/점수 입력 폼이 한 손으로 조작 가능하도록.
